## Create Your First Rails-like API

Let's build a simple blog API to demonstrate Zenith's Rails-inspired features.

1. **Install Zenith**

   ```bash
   pip install zenith-web
   ```

2. **Create your application file**

   Create `app.py`:

   ```python
   from zenith import Zenith
   from zenith import Session  # Enhanced dependency injection
   from zenith.db import ZenithModel  # Rails-like models
   from sqlmodel import Field
   from typing import Optional

   # üöÄ Zero-config setup - just works!
   app = Zenith()

   # ‚ö° Add features in one line each
   app.add_auth()    # JWT authentication + /auth/login
   app.add_admin()   # Admin dashboard at /admin
   app.add_api("Blog API", "1.0.0")  # API docs at /docs

   # üèóÔ∏è Rails-like model with ActiveRecord patterns
   class Post(ZenithModel, table=True):
       id: Optional[int] = Field(primary_key=True)
       title: str = Field(max_length=200)
       content: str
       author: str = Field(max_length=100)
       published: bool = Field(default=False)

   @app.get("/")
   async def root():
       return {
           "message": "Welcome to Rails-like Blog API",
           "features": ["Rails-like models", "One-liner setup", "Enhanced DX"],
           "docs": "/docs",
           "admin": "/admin"
       }

   @app.get("/posts")
   async def get_posts(published: bool = True, session=Session):
       """Get blog posts with Rails-like queries."""
       if published:
           # Rails-style: Post.where(published=True).order_by('-id').limit(10)
           query = await Post.where(published=True)
           posts = await query.order_by('-id').limit(10).all()
       else:
           posts = await Post.all()
       return {"posts": [post.to_dict() for post in posts]}

   @app.post("/posts")
   async def create_post(post_data: dict, session=Session):
       """Create post with Rails-style convenience."""
       # Rails-style: Post.create() - no session management!
       post = await Post.create(**post_data)
       return {"post": post.to_dict(), "message": "Post created"}

   @app.get("/posts/{post_id}")
   async def get_post(post_id: int, db=DB):
       """Get post with automatic 404 handling."""
       # Rails-style: automatic 404 if not found
       post = await Post.find_or_404(post_id)
       return {"post": post.to_dict()}

   @app.put("/posts/{post_id}")
   async def update_post(post_id: int, updates: dict, db=DB):
       """Update post with Rails-style convenience."""
       post = await Post.find_or_404(post_id)
       # Rails-style: model.update() method
       await post.update(**updates)
       return {"post": post.to_dict(), "message": "Post updated"}

   @app.delete("/posts/{post_id}")
   async def delete_post(post_id: int, db=DB):
       """Delete post with Rails-style convenience."""
       post = await Post.find_or_404(post_id)
       # Rails-style: model.destroy() method
       await post.destroy()
       return {"message": "Post deleted successfully"}
   
   # Health check endpoint
   @app.get("/health")
   async def health_check():
       return {
           "status": "healthy",
           "timestamp": datetime.utcnow(),
           "version": app.version
       }
   ```

3. **Run the development server**

   ```bash
   python app.py
   # Or use uvicorn directly
   uvicorn app:app --reload
   ```

4. **Test your Rails-like API**

   Your API is now running at `http://localhost:8000` with these features:
   - üìö **API Docs**: http://localhost:8000/docs (Interactive Swagger UI)
   - üîê **Authentication**: POST to http://localhost:8000/auth/login
   - ‚ö° **Admin Dashboard**: http://localhost:8000/admin
   - üè• **Health Check**: http://localhost:8000/health

**Test the Rails-like CRUD:**

<Aside type="tip" title="Rails-like DX Benefits">
Compare this Zenith code to equivalent FastAPI + SQLAlchemy:

- **85% less boilerplate** - No manual session management, dependency setup, or middleware configuration
- **Zero-config setup** - `app = Zenith()` vs 50+ lines of FastAPI setup
- **Rails-like queries** - `Post.where().order_by().limit()` vs complex SQLAlchemy
- **One-liner features** - `app.add_auth()` vs 20+ lines of auth setup
- **Automatic 404s** - `Post.find_or_404()` vs manual error handling
</Aside>
